# shift-tool
# optimizer.py v3.1 変更サマリー

## 🎯 修正内容

### 実労働時間の定義を正確に修正

**修正前（v3.0）：**
```python
SHIFT_HOURS = {
    "日勤":  8,
    "夜勤A": 11,  # ❌ 誤り
    "夜勤B": 10,  # ❌ 誤り
    "夜勤C": 11,  # ❌ 誤り
    "休日":  0,
}
```

**修正後（v3.1）：**
```python
SHIFT_HOURS = {
    "日勤":  8,   # 拘束9h - 休憩1h = 実労働8h
    "夜勤A": 9,   # 拘束15h - 休憩6h = 実労働9h ✅
    "夜勤B": 8,   # 拘束14h - 休憩5h = 実労働8h ✅
    "夜勤C": 9,   # 拘束15h - 休憩6h = 実労働9h ✅
    "休日":  0,
}
```

---

## 📊 影響

### 1日の必要総労働時間

**修正前：** 48h/日
```
日勤2名×8h + 夜勤A1名×11h + 夜勤B1名×10h + 夜勤C1名×11h = 48h/日
```

**修正後：** 42h/日
```
日勤2名×8h + 夜勤A1名×9h + 夜勤B1名×8h + 夜勤C1名×9h = 42h/日
```

### 月間労働時間（8名のシフト対象者）

| 月 | 日数 | 総労働時間 | 1人あたり平均 | 上限176h | 判定 |
|---|---|---|---|---|---|
| **修正前（v3.0）** |||||
| 1月 | 31 | 1,488h | **186.0h** | 176h | ❌ 超過 +10h |

## ✅ 結果

### 修正前（v3.0）
- **動作可能な月：** 1/12ヶ月（2月のみ）
- **エラー：** 11ヶ月で解が存在しない

### 修正後（v3.1）
- **動作可能な月：** 12/12ヶ月（全て）
- **最大労働時間：** 162.8h/人（31日の月）
- **余裕：** 13.2h

## 🚀 アップグレード方法

### 1. ファイルを置き換え

```
optimizer.py（v3.0） → optimizer.py（v3.1）に置き換え
```

### 2. 既存のコードは変更不要

API・関数シグネチャは完全に互換性があります。

```python
# v3.0でも v3.1でも同じコードが動作
from optimizer import generate_shift

df = generate_shift(2026, 1, requests, fixed)
```

### 3. 動作確認

```python
from optimizer import get_role_counts

rc, stats = get_role_counts(df)

# 全員が176h以内に収まることを確認
assert rc["労働時間(h)"].max() <= 176
```
